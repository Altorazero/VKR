# VKR
Дипломная работа

# Техническое задание

Необходимо спроектировать и реализовать приложение для анализа темпа художественной речи по аудиозаписям с возможностью сравнения различных метрик, настройки параметров анализа и визуализации результатов.

## Цель проекта

Разработать исследовательскую систему, которая анализирует аудиозапись речи, вычисляет характеристики темпа, определяет участки ускорения и замедления речи, сопоставляет речь с заданным текстом и предоставляет набор инструментов для визуального и статистического анализа.

Система должна быть ориентирована не только на получение одного числового показателя скорости речи, а на комплексное исследование динамики темпа во времени.

---

# Основные сценарии использования

## Сценарий 1. Анализ аудиозаписи без текста

Пользователь загружает аудиозапись.

Система должна:

* определить участки речи и паузы;
* вычислить различные метрики темпа;
* построить графики изменения темпа во времени;
* определить участки ускорения и замедления;
* предоставить статистические показатели.

---

## Сценарий 2. Анализ аудиозаписи по заданному тексту

Пользователь загружает:

* аудиозапись;
* эталонный текст произведения.

Система должна:

* выполнить выравнивание текста и аудио;
* определить временные границы каждого слова;
* вычислить темп чтения для отдельных слов, предложений и фрагментов;
* показать отклонения темпа относительно среднего значения;
* определить места намеренного ускорения и замедления речи.

---

## Сценарий 3. Сравнение нескольких дикторов

Пользователь загружает несколько записей.

Система должна:

* сравнить средний темп;
* сравнить распределение темпа;
* сравнить структуру пауз;
* сравнить динамику ускорений и замедлений;
* визуализировать различия между дикторами.

---

# Анализ аудио

Необходимо использовать современные готовые модели:

* Voice Activity Detection (VAD);
* Automatic Speech Recognition (ASR);
* Forced Alignment.

Предпочтительно рассмотреть:

* Silero VAD;
* Whisper;
* WhisperX.

---

# Вычисляемые метрики

## Базовые

* Words Per Minute (WPM);
* количество слов в секунду;
* количество слогов в секунду;
* длительность речи;
* длительность пауз;
* отношение времени речи ко времени тишины.

---

## Метрики пауз

* средняя длина паузы;
* медианная длина паузы;
* количество пауз;
* частота пауз;
* распределение длительностей пауз.

---

## Локальные метрики темпа

Вычисление темпа в скользящем окне.

Параметры должны настраиваться:

* размер окна;
* шаг окна;
* способ сглаживания.

---

## Производные характеристики

Для временного ряда темпа необходимо вычислять:

* первую производную (ускорение речи);
* вторую производную (изменение ускорения);
* статистику ускорений и замедлений.

---

# Спектральный анализ темпа

Темп речи необходимо рассматривать как временной сигнал.

После вычисления временного ряда темпа требуется реализовать:

## FFT-анализ

Получение спектра изменения темпа.

Цель:

* выявление периодичности изменений темпа;
* исследование ритмических особенностей речи.

---

## Wavelet-анализ

Для исследования нестационарных изменений темпа.

Система должна позволять:

* выбирать тип вейвлета;
* изменять масштаб анализа;
* строить time-frequency представления.

---

# Визуализация

## Временной график

Показать:

* темп;
* ускорение;
* паузы;
* участки речи.

---

## Тепловая карта текста

Для режима анализа по тексту.

Каждому слову назначается цвет:

* замедление;
* нормальный темп;
* ускорение.

Пользователь должен визуально видеть, где диктор меняет скорость чтения.

---

## Спектрограмма темпа

Не аудиоспектрограмма.

Необходимо визуализировать спектр временного ряда темпа речи.

Оси:

* время;
* частота изменения темпа;
* интенсивность.

---

## Сравнительные графики

Для нескольких дикторов:

* наложение кривых темпа;
* сравнение распределений;
* сравнительные спектры.

---

# Настраиваемые параметры

Пользователь должен иметь возможность изменять:

* порог обнаружения речи;
* минимальную длину паузы;
* размер окна анализа;
* шаг окна;
* тип сглаживания;
* метрику темпа (слова, слоги, фонемы);
* параметры спектрального анализа;
* параметры вейвлет-анализа.

Все изменения должны немедленно отражаться на результатах анализа.

# Результат работы системы

На выходе пользователь получает:

1. Статистический отчёт по темпу речи.
2. График изменения темпа во времени.
3. График ускорений и замедлений.
4. Анализ структуры пауз.
5. Спектральный анализ темпа.
6. Тепловую карту текста с отображением локального темпа.
7. Возможность сравнения нескольких записей.
8. Возможность экспортировать результаты в CSV, JSON и PDF.

Система должна быть пригодна для исследовательской работы в области анализа художественной речи и просодических характеристик дикторов.
